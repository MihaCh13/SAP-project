# 🏢 SAP Secure Document Manager

Надеждна Enterprise система за управление на документи (DMS), изградена с Java и Spring Boot. Проектът включва сигурен бекенд с REST API и динамичен, интерактивен конзолен клиент (CLI) за крайните потребители.

## 🚀 Ключови функционалности

* **Ролев модел за достъп (RBAC):** Строга сигурност, базирана на роли: `ADMIN`, `AUTHOR`, `REVIEWER` и `READER`.
* **Управление на работните процеси (Workflow):** Документите следват стриктен жизнен цикъл: `DRAFT` ➔ `PENDING_REVIEW` ➔ `APPROVED` или `REJECTED`.
* **Контрол на версиите:** Пълно проследяване на историята на документите и направените редакции (неизменяеми версии).
* **Система за известия (Notifications):** Асинхронно известяване на потребителите в реално време при промяна на системните им роли, одобрение или отхвърляне на техни документи.
* **Експортиране на файлове:** Генериране на професионални `.pdf` и `.txt` отчети за финалните одобрени версии на документите.
* **Динамичен конзолен клиент:** Интелигентен клиент, който адаптира менюто си спрямо правата и ролите на успешно логнатия потребител.

## 🛠️ Използвани технологии

* **Бекенд:** Java, Spring Boot 3.2, Spring Web, Spring Data JPA
* **База данни:** H2 (In-Memory Database) с Hibernate ORM
* **Клиент:** Core Java (`java.net.http.HttpClient` за комуникация с API-то)
* **Библиотеки:** `OpenPDF` (за генериране на PDF документи)

## 🏃‍♂️ Как да стартирате проекта

1. **Стартиране на Сървъра (Backend):** Стартирайте главния Spring Boot клас (`Main.java`). Базата данни H2 ще се инициализира автоматично и ще зареди първоначалните роли и потребители чрез `DataInitializer`. Сървърът ще тръгне на порт `8080`.
2. **Стартиране на Клиента (Console UI):** Стартирайте метода `main` в класа `ConsoleClient.java`. В конзолата ще се отвори интерактивното меню за вход в системата.
3. **Достъп до базата данни (H2 Console):**
   Можете да разгледате създадените таблици директно през браузъра на адрес: `http://localhost:8080/h2-console`
    * *JDBC URL:* `jdbc:h2:mem:sapdb`
    * *User:* `SA` (Паролата се оставя празна)

## 🧪 Тестови акаунти

За бързо стартиране и тестване, системата автоматично генерира следните потребители (паролата за всеки е името му + `123`, напр. `admin123`):

| Потребител (Username) | Парола (Password) | Роли |
| :--- | :--- | :--- |
| **admin** | `admin123` | `ADMIN` |
| **author** | `author123` | `AUTHOR` |
| **reviewer** | `reviewer123` | `REVIEWER` |
| **reader** | `reader123` | `READER` |
| **lead_author** | `lead123` | `AUTHOR`, `REVIEWER` |
| **super_user** | `super123` | `ADMIN`, `AUTHOR`, `REVIEWER` |
