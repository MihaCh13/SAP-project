# 🚀 Документация на API Слоя (Controller Layer)

Този документ описва работата на `com.sap.project.controllers.DocumentController`. Този компонент е "мостът" между външния свят и вътрешната логика на нашето приложение.

---

## 1. Роля на Контролера
Контролерът приема заявки от потребителите, обработва входните данни и координира работата между базата данни на Петьо (`Repositories`) и бизнес правилата на Михаела (`Services`).

---

## 2. Ключови Ендпоинти (Endpoints)

### 📜 Извличане на История (`getDocumentHistory`)
Този метод позволява на потребителя да види пътя на един документ във времето.
- **Вход:** `Integer docId` (Уникален идентификатор на документа).
- **Логика:** Използва директно `VersionRepository` за извличане на всички версии, свързани с този документ.
- **Резултат:** Връща списък от `VersionEntity`, съдържащ пълната метаинформация.

### ✅ Одобрение на Версия (`approveVersion`)
Най-критичната функция, която задвижва работния процес (Workflow).
- **Вход:** `versionId` и текстови коментар.
- **Процес:** 1. Намира суровите данни в базата чрез `versionRepository`.
    2. **Трансформация:** Превръща `Entity` обектите в `Business Models` (`User`, `Version`, `Document`), за да може бизнес логиката да ги разбере.
    3. **Валидация:** Симулира роля `REVIEWER` за потребителя, който извършва действието.
    4. **Изпълнение:** Извиква `workflowService.approveDocument` за финализиране на процеса.

---

## 🏗️ Интеграция с другите слоеве

| Слой | Компонент | Какво правим с него? |
| :--- | :--- | :--- |
| **Persistence** | `VersionRepository` | Използваме го за четене на текущото състояние от DB. |
| **Business** | `WorkflowService` | Делегираме му тежката логика по одобрение и проверка на статуси. |
| **Models** | `Document`, `Version`, `User` | Създаваме тези обекти "в движение", за да захраним логиката на Михаела. |

---

## 💡 Важно за разработчиците
В момента контролерът ръчно мапва (превръща) `VersionEntity` в `Version` модел. Това гарантира, че ако Петьо промени нещо в таблиците, ние можем да го коригираме тук, без да чупим кода на Михаела.
