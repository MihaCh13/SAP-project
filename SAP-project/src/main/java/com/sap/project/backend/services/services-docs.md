# Пакет: com.sap.project.backend.services

Този пакет представлява сърцето на бизнес логиката в системата.  
Той е разделен на два основни компонента, които гарантират сигурността, правилния работен процес на документите и стриктното управление на потребителските права.

---

## 1. WorkflowService.java

**WorkflowService** управлява жизнения цикъл на документите и техните версии.  
Действа като мост между действията на потребителите и състоянието на данните в базата.

### Основни функционалности

#### 1. Създаване на документ (`createDocument`)
- Позволява само на потребители с роля `AUTHOR` да създават нови записи.
- Автоматично инициира първата версия (V1) със статус `DRAFT`.

#### 2. Редактиране на документ (`editDocument`)
- Позволява на авторите да създават нови версии (V2, V3…) към съществуващ документ.
- **Collaboration:** Авторите могат да редактират документи взаимно, независимо кой е оригиналният създател.

#### 3. Изпращане за преглед (`submitForReview`)
- Проверява дали потребителят е автор на конкретната версия.
- Само собственикът на редакцията може да я придвижи към статус `PENDING_REVIEW`.

#### 4. Одобряване и отхвърляне (`approveDocument` / `rejectDocument`)
- Достъпно само за потребители с роля `REVIEWER`.
- **Защита от конфликт на интереси:** Рецензентът не може да обработва версии, които сам е създал/редактирал.
- Одобрената версия автоматично става "Активна" за документа.

#### 5. Четене на версия (`viewVersion`)
- Гарантира, че потребители с роля `READER` виждат само официално одобрени версии (`APPROVED`).

### Примерен код

```java
WorkflowService workflowService = new WorkflowService();

// Създаване на нов документ
Document doc = workflowService.createDocument(authorUser, 101, "План 2026", "Годишен план", "Съдържание...");

// Редакция (Създаване на нова версия)
Version v2 = workflowService.editDocument(anotherAuthor, doc, 202, 2, "Обновено съдържание", 201);

// Изпращане за преглед
workflowService.submitForReview(anotherAuthor, v2);

// Одобряване от рецензент
workflowService.approveDocument(reviewerUser, v2, doc);

// Отхвърляне от рецензент
workflowService.rejectDocument(reviewerUser, v2);

// Четене на версия
workflowService.viewVersion(readerUser, v2);
```

## 🔑 2. UserService.java

**UserService** отговаря за административните задачи и сигурността на потребителските акаунти.  
Достъпът до тези методи е строго ограничен само за потребители с роля `ADMIN`.

---

### 🛠️ Административни функции

#### 1. Управление на роли (`assignRole` / `revokeRole`)
- Администраторът може динамично да **добавя** или **премахва** права на други потребители.  
- **Admin Lockout Protection:** Системата не позволява на администратор да премахне собствената си `ADMIN` роля, за да не загуби достъп.

#### 2. Управление на достъпа (`activateUser` / `deactivateUser`)
- Възможност за **спиране** или **възстановяване** на достъпа до системата.  
- Администраторът е защитен от деактивиране на собствения си акаунт.

---

### 💻 Примерен код за Администриране

```java
UserService userService = new UserService();

// Промяна на права
userService.assignRole(adminUser, targetUser, Role.AUTHOR);

// Премахване на роля
userService.revokeRole(adminUser, targetUser, Role.AUTHOR);

// Деактивиране на акаунт при напускане на служител
userService.deactivateUser(adminUser, formerEmployee);

// Активиране на потребител
userService.activateUser(adminUser, targetUser);
```

| Изключение                 | Кога се хвърля                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------------------- |
| `SecurityException`        | При опит за действие без необходимите права или при конфликт на интереси.                       |
| `IllegalArgumentException` | При подаване на невалидни данни (null обекти, празни заглавия или отрицателни ID-та).           |
| `IllegalStateException`    | При опит за неправилен преход между статуси (напр. одобрение на версия, която не е за преглед). |
