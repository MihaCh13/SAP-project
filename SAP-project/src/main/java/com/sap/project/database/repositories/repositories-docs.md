# 🗄️ Документация на Репозитория (Data Access Layer)

Пакетът `com.sap.project.database.repositories` съдържа интерфейсите, които управляват директната комуникация с базата данни. Те разширяват `JpaRepository`, което предоставя вградени методи за CRUD операции (Create, Read, Update, Delete) без нужда от писане на ръчен SQL.

---

## 1. Обща концепция

Репозиториите работят директно с **Database Entities** и се използват от **Services** слоя чрез Dependency Injection. Всички методи тук се изпълняват в рамките на трансакция, осигурявайки интегритет на данните.



---

## 2. Описание на интерфейсите

### 👤 UserRepository
Отговаря за управлението на потребителските данни.
- **Вградени методи:** `save`, `findById`, `delete`.
- **Custom Query Methods:**
    - `findByUsername(String username)`: Търси потребител по неговото име (връща `Optional`).
    - `findByEmail(String email)`: Търси потребител по имейл адрес (връща `Optional`).
- **Употреба:** Използва се от `UserService` за проверка на съществуващи акаунти и управление на роли.

### 📄 DocumentRepository
Управлява метаданните на документите.
- **Тип:** `JpaRepository<DocumentEntity, Integer>`.
- **Цел:** Служи за иницииране на нови документи и промяна на техните глобални статуси (напр. архивиране чрез `isActive` флага).

### 📑 VersionRepository
Управлява съдържанието и историята на версиите. Това е най-често използваното репозитори при работа с документи.
- **Custom Query Methods:**
    - `findByDocumentId(Integer documentId)`: Извлича пълната история на всички версии за конкретен документ, подредени по тяхната последователност.
- **Употреба:** Критично за `WorkflowService` при сравняване на версии и визуализиране на историята.

---

## 🛠️ Технически детайли за екипа

| Функция | Описание |
| :--- | :--- |
| **Технология** | Spring Data JPA (Hibernate под капака) |
| **База данни** | H2 (InMemory за разработка) / SQL |
| **Query Logic** | Използва се *Query derivation* (заявките се генерират автоматично от името на метода) |
| **Return Types** | Използват се `Optional<T>` за единични записи и `List<T>` за множество записи |
