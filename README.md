# SAP-project
Document Version Control System (DVCS) designed for collaborative document management. Features include role-based access (Author, Reviewer, Reader, Admin), version history tracking, approval workflows, and client-server architecture. Developed as part of a SAP course project.

# Google Gemini

## 📝 Описание на проекта

В много организации се работи с документи, които претърпяват чести промени от множество потребители. Липсата на контрол води до загуба на информация и объркване относно това коя е последната актуална версия.

Този проект представлява софтуерно решение за управление на версии на документи, което гарантира проследимост, сигурност и правилен работен процес (workflow) чрез роли и одобрения.

---

## 🏗️ Архитектура и технически изисквания

- **Архитектура:** Клиент-сървър (Client-Server), поддържаща едновременна работа на множество потребители.  
- **Език за разработка:** Java  
- **Формат на данните:** SQL база данни / JSON / XML  
- **Интерфейс:** Конзолен режим (планиран GUI като бонус)  
- **Стабилност:** Пълно обхващане на грешки (Exception Handling) и съобщаване при невалидни действия или липса на права  

---

## 👥 Потребителски роли

| Роля        | Права и възможности |
|------------|-------------------|
| **Автор**      | Създава нови документи и версии, редактира чернови и преглежда историята |
| **Рецензент**  | Одобрява или отхвърля версии, добавя коментари към документите |
| **Читател**    | Достъп само за четене на активните и вече одобрени версии |
| **Администратор** | Управлява потребителските профили, техните роли и конфигурацията на системата |

---

## 🔥 Основни функционалности

1. **Линейно версиониране:** Всеки документ съдържа метаданни и последователност от версии  
2. **Неизменяемост:** Веднъж създадена версия не може да бъде променяна (Immutable)  
3. **Работен процес за одобрение:** Нова версия става "Активна" само след статус `APPROVED`. Отхвърлените версии остават в историята  
4. **Сравнение:** Функционалност за сравняване на разликите между две версии на един и същ документ  
5. **Сигурност:** Валидация на правата преди всяка операция
