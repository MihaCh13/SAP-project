# SAP-project
Document Version Control System (DVCS) designed for collaborative document management. Features include role-based access (Author, Reviewer, Reader, Admin), version history tracking, approval workflows, and client-server architecture. Developed as part of a SAP course project.

Система за управление на версии на документи (DVCS)

📝 Описание на проекта

В много организации се работи с документи, които претърпяват чести промени от множество потребители. Липсата на контрол води до загуба на информация и объркване относно това коя е последната актуална версия.

Този проект представлява софтуерно решение за управление на версии на документи, което гарантира проследимост, сигурност и правилен работен процес (workflow) чрез роли и одобрения.

🏗️ Архитектура и Технически изисквания

Системата е изградена на базата на следните принципи и технологии:

Архитектура: Клиент-сървър (Client-Server), поддържаща едновременна работа на множество потребители.

Език за разработка: Java.

Формат на данните: SQL База данни (като основен избор) / JSON / XML.

Интерфейс: Интерактивен конзолен режим (с планиран GUI като бонус).

Стабилност: Пълно обхващане на грешките (Exception Handling) и съобщаване при невалидни действия или липса на права.

👥 Потребителски роли

Системата дефинира четири нива на достъп:

Роля

Права и възможности

Автор

Създава нови документи и версии, редактира чернови и преглежда историята.

Рецензент

Одобрява или отхвърля версии, добавя коментари към документите.

Читател

Достъп само за четене на активните и вече одобрени версии.

Администратор

Управлява потребителските профили, техните роли и конфигурацията на системата.

🔥 Основни функционалности

Линейно версиониране: Всеки документ съдържа метаданни и последователност от версии.

Неизменяемост: Веднъж създадена и записана, версията не може да бъде променяна (Immutable).

Работен процес за одобрение: Нова версия става "Активна" само след статус APPROVED. Отхвърлените версии остават в историята за справка.

Сравнение: Функционалност за сравняване на разликите между две версии на един и същ документ.

Сигурност: Валидация на правата преди всяка операция.
